## Design Context

### Users
이 앱의 주요 사용자는 학교 과학정보부 담당교사와 내부 운영 담당자입니다. 사용 맥락은 수업 직전 네트워크 장애 대응, 사이트 계정 정리, 정보화기기대장 점검처럼 시간이 부족하고 실수가 바로 업무 지연으로 이어지는 상황입니다. 가장 중요한 일은 빠르게 찾고, 바로 판단하고, 즉시 수정하는 것입니다.

### Brand Personality
차분함, 신뢰감, 정확성. 화면은 친절해야 하지만 가볍거나 장식적으로 보이면 안 됩니다. 사용자는 이 앱을 홍보 페이지가 아니라 학교 내부 운영 도구로 인식해야 하며, 긴급 상황에서도 망설임 없이 사용할 수 있어야 합니다.

### Aesthetic Direction
기본 방향은 밝은 라이트 테마의 공공 업무도구형 UI입니다. IBM 계열의 정돈된 엔터프라이즈 느낌을 바탕으로 하고, Airtable처럼 표와 필터의 가독성을 우선합니다. 주요 기준은 Paperlogy 중심의 한국어 가독성, 옅은 회색 배경, 흰 표면, 남색 계열 포인트, 절제된 그림자입니다. 피해야 할 방향은 랜딩페이지형 히어로 배너, 과한 카드 장식, 보라색 중심 AI 스타일, 과장된 그래디언트와 유리 질감입니다.

### Accessibility & Readability
접근성 기준은 내부 업무용 환경에 맞는 실무형 사용성을 목표로 합니다. 최소한 WCAG AA 수준의 가독성을 지향하며, 본문은 14px 이상, 작은 보조 텍스트도 13px 이상을 유지합니다. 상태 정보는 색만으로 구분하지 않고 반드시 라벨을 함께 사용합니다. 버튼과 입력 요소는 키보드 탭 이동과 포커스 표시가 분명해야 하며, 과한 애니메이션은 피합니다. 이 기준을 적용한 결과 레이아웃이 깨지거나 화면 구조가 무너지면, 접근성 규칙을 유지한 채 레이아웃을 다시 조정해야 합니다. 기본 화면에서는 가로 스크롤을 사용하지 않는 것을 목표로 하며, 한글 특성을 고려해 어색한 단어 분리보다 자연스러운 줄바꿈과 띄어쓰기를 우선합니다.

### Design Principles
1. 검색과 상태 확인은 항상 화면 상단에서 가장 먼저 보여야 합니다.
2. 표는 표답게 유지하고, 실제 업무 데이터는 숨기지 않습니다.
3. 한글 문장은 짧고 자연스럽게 줄바꿈되며, 식별자는 굵고 분명하게 구분합니다.
4. 위험 상태와 작업 가능 상태는 색과 라벨로 즉시 구분되게 합니다.
5. 장식보다 정렬, 간격, 위계, 반복 규칙의 일관성을 우선합니다.
6. 기본 화면은 가로 스크롤 없이 읽히도록 구성하고, 좁은 폭에서는 레이아웃을 재배치해도 정보 우선순위는 유지합니다.
