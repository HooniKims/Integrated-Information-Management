# 기기관리대장 XLSX 대시보드형 보고서 설계

작성일: 2026-04-21

## 1. 목적

현재 기기관리대장 XLSX 다운로드는 표 중심의 단순 내보내기 형식이다. 이를 `다운로드한 즉시 보고서처럼 보이는 엑셀`로 개선하되, 학교 내부 운영 도구의 성격과 실무 사용성을 유지하는 것을 목표로 한다.

이번 설계의 핵심은 다음과 같다.

- 첫 시트는 보고용 대시보드로 보이게 만든다.
- 둘째 시트는 전체 데이터를 빠르게 확인할 수 있는 원본 대장으로 둔다.
- 이후 시트는 설치장소별 탭으로 자동 분리해 실무 탐색 속도를 높인다.
- 디자인은 화려함보다 정돈감, 신뢰감, 읽기 쉬움을 우선한다.

## 2. 설계 원칙

- 내부 운영 도구답게 `밝은 배경 + 네이비 포인트 + 회색 구조선` 중심으로 정리한다.
- 차트와 카드가 있어도 `보고서형 엑셀` 범위를 벗어나지 않게 한다.
- 데이터는 숨기지 않는다. 첫 시트는 요약, 둘째 시트 이후는 즉시 실무 확인 가능해야 한다.
- 장소명이 있는 데이터는 자동으로 탭을 생성하되, 시트명이 너무 길거나 엑셀 규칙에 맞지 않는 경우 안전하게 정규화한다.
- 사용자가 다운로드했을 때 `첫인상`, `탐색 속도`, `관리 편의성`이 동시에 좋아져야 한다.

## 3. 최종 시트 구조

워크북은 아래 순서로 생성한다.

1. `대시보드`
2. `전체대장`
3. 설치장소별 시트들

설치장소별 시트는 현재 데이터에 존재하는 설치장소 값을 기준으로 자동 생성한다.

예시:

- `대시보드`
- `전체대장`
- `컴퓨터실`
- `교무실`
- `방송실`
- `과학실`

설치장소가 비어 있는 항목은 별도 예외 탭을 두기보다 `미지정` 같은 안전한 이름으로 묶는다.

## 4. 대시보드 시트 구성

첫 시트 `대시보드`는 `깔끔한 보고서형`으로 구성한다.

### 4.1 상단 제목 영역

- 메인 제목: `DCMS 기기관리대장`
- 부제: `설치장소별 자산 분포 요약`
- 우측 정보:
  - 생성일
  - 전체 장비 수
  - 설치장소 수

이 영역은 병합 셀과 배경색 블록을 활용해 일반 표와 확실히 구분되도록 만든다.

### 4.2 요약 카드 영역

상단 제목 아래에는 3~4개의 요약 카드를 둔다.

- 전체 장비 수
- 설치장소 수
- 가장 많은 장비가 있는 장소
- 정상 사용 장비 수 또는 점검 필요 장비 수

카드는 `진한 제목 + 큰 숫자 + 작은 설명` 구조로 통일한다.

### 4.3 메인 분포 영역

대시보드의 중심은 `설치장소별 보유 수량`이다.

- 장소별 장비 수를 내림차순으로 정렬한 표를 만든다.
- 같은 데이터를 기반으로 평면형 막대 차트를 함께 배치한다.
- 장소 수가 많으면 상위 N개만 차트에 보여주고, 나머지는 표에서 전체 확인 가능하게 한다.

권장 기준:

- 차트: 상위 8개 장소
- 표: 전체 장소

### 4.4 보조 요약 영역

하단 또는 우측에는 아래 보조 정보를 둔다.

- 상태별 분포
  - 정상 사용
  - 점검 필요
  - 교체 검토
- 기기형태별 간단 요약
- 안내 문구
  - `장소별 상세 목록은 각 시트 탭에서 확인`

## 5. 전체대장 시트 구성

`전체대장` 시트는 기존 대장 역할을 유지하되, 시각 완성도를 높인다.

### 5.1 표 구조

- 현재 `csv_headers` 기준 컬럼 구조를 유지한다.
- 자동필터를 유지한다.
- 첫 행 고정을 유지한다.
- 줄바꿈과 세로 정렬을 정돈한다.

### 5.2 시각 스타일

- 헤더는 네이비 또는 옅은 블루 계열 배경으로 통일한다.
- 데이터 행은 줄무늬 배경을 적용해 읽기 쉽게 만든다.
- 상태 컬럼은 값에 따라 강조색을 다르게 준다.
  - 정상 사용: 차분한 블루 또는 그린
  - 점검 필요: 앰버
  - 교체 검토: 레드 계열

### 5.3 열 너비

- 현재 단순 `len(title)` 기반보다 조금 더 실용적으로 조정한다.
- 긴 텍스트 컬럼은 넓게, 짧은 코드 컬럼은 좁게 배치한다.

권장 대상:

- 넓게: `모델명`, `비고`, `사용자명`, `제품이미지`
- 좁게: `분류`, `형태`, `RAM`, `사용연수`, `상태`

## 6. 설치장소별 시트 구성

설치장소별 시트는 `전체대장`과 같은 컬럼 구조를 유지하되, 해당 장소 데이터만 담는다.

### 6.1 시트 상단 정보

각 장소 시트 상단에는 간단한 헤더를 둔다.

- 시트 제목: 해당 설치장소명
- 장비 수
- 생성일

### 6.2 데이터 영역

- 해당 장소 데이터만 표로 출력한다.
- 헤더 스타일은 `전체대장`과 동일하게 유지한다.
- 첫 행 고정, 자동필터를 동일하게 적용한다.

### 6.3 사용성

- 사용자는 대시보드에서 장소 분포를 본 뒤, 바로 해당 장소 탭으로 들어가 상세 데이터를 확인할 수 있어야 한다.
- 장소 탭은 생성 순서보다 `장비 수가 많은 장소 순` 또는 `이름순` 중 하나로 고정해야 한다.

권장안:

- 이름순 정렬

이유:

- 사용자가 시트 탭을 찾기 쉽다.

## 7. 엑셀 스타일 가이드

### 7.1 색상

- 메인 포인트: 네이비
- 보조 배경: 옅은 블루/그레이
- 경계선: 연한 회색
- 강조색:
  - 정상: 차분한 그린 또는 블루
  - 주의: 앰버
  - 위험: 레드

### 7.2 폰트와 정렬

- 엑셀 기본 폰트를 유지하되, 제목/숫자 강조에 크기 차이를 둔다.
- 큰 제목은 굵게, 요약 숫자는 더 크게, 본문 표는 과하지 않게 유지한다.
- 가운데 정렬은 제목과 숫자 카드에만 제한적으로 사용하고, 실데이터는 기본적으로 읽기 편한 정렬을 적용한다.

### 7.3 장식 제한

- 3D 차트, 과한 그라데이션, 그림자 남용은 피한다.
- 병합 셀은 제목/카드 영역까지만 제한적으로 사용한다.
- 어디까지나 `내부 운영 보고서`처럼 보여야 한다.

## 8. 데이터 처리 규칙

### 8.1 장소명 정규화

설치장소 값은 시트명으로도 사용되므로 아래 규칙이 필요하다.

- 공백만 있는 값은 `미지정`으로 처리
- 엑셀 시트명 제한 문자는 제거 또는 치환
- 시트명 길이 제한을 넘지 않도록 자른다
- 중복 시트명이 생기면 접미사를 붙여 충돌을 피한다

### 8.2 분포 집계

대시보드 집계는 `list_devices()` 결과를 기준으로 계산한다.

- 설치장소별 수량
- 상태별 수량
- 기기형태별 수량

### 8.3 상태 분류

기존 데이터의 `status`, `life_cycle_due`, `repair_or_inspection_needed`를 활용해 요약한다.

## 9. 구현 범위

이번 구현 범위에 포함한다.

- `대시보드` 시트 추가
- `전체대장` 시트 스타일 개선
- 설치장소별 시트 자동 생성
- 장소별 분포 표 및 차트 추가
- 요약 카드와 상태 요약 추가

이번 구현 범위에서 제외한다.

- 사용자 정의 템플릿 편집 UI
- 차트 종류를 웹에서 선택하는 기능
- 엑셀 안의 고급 피벗테이블/매크로
- 인쇄 최적화 전용 레이아웃 튜닝

## 10. 검증 기준

구현 후 아래 기준을 확인한다.

- XLSX 파일이 정상 다운로드된다.
- 첫 시트가 `대시보드`로 열린다.
- `전체대장`과 장소별 탭이 모두 생성된다.
- 장소별 탭 수가 실제 설치장소 집계와 맞는다.
- 차트와 요약 숫자가 데이터와 일치한다.
- 긴 장소명, 빈 장소명, 중복 장소명에서도 시트 생성이 깨지지 않는다.
- `openpyxl` 저장 후 Excel에서 열었을 때 경고가 없어야 한다.

## 11. 추천 구현 순서

1. 장소별 집계 함수 추가
2. 공통 스타일 유틸 정리
3. `대시보드` 시트 생성
4. `전체대장` 시트 스타일 개선
5. 설치장소별 시트 자동 생성
6. 테스트 보강

## 12. 기대 효과

기존 보고서가 `단순 표 다운로드`였다면, 개선 후에는 아래와 같은 인상을 주게 된다.

- 첫 탭은 보고용 요약 자료
- 둘째 탭은 전체 관리용 실무 대장
- 이후 탭은 장소별 탐색용 세부 대장

즉, 한 파일 안에서 `보고`, `확인`, `탐색`이 나뉜 구조가 되어 훨씬 완성도 있는 내부 운영 문서처럼 보이게 된다.
